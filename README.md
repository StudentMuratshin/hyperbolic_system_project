# Сравнение схем Русанова, HLL и Roe для уравнений мелкой воды

Проект реализует конечно-объемный метод для одномерной гиперболической системы уравнений мелкой воды и сравнивает три приближенных решателя задачи Римана:

- Rusanov flux;
- HLL flux;
- Roe flux с простой энтропийной поправкой.

Модель записана в консервативной форме:

```text
U_t + F(U)_x = 0,

U = (h, hu)^T,
F(U) = (hu, hu^2 / h + g h^2 / 2)^T.
```

Собственные значения матрицы Якоби:

```text
lambda_1 = u - sqrt(g h),
lambda_2 = u + sqrt(g h).
```

Они действительные при `h > 0`, поэтому система гиперболическая.

## Что делает программа

После запуска программа создает каталог `results/` и записывает CSV-файлы:

- `profile_<case>_<method>.csv` - финальные профили `h(x)`, `u(x)`, `hu(x)`;
- `mass_<case>_<method>.csv` - история полной массы;
- `summary.csv` - ошибки, время работы, сохранение массы, ширина правого фронта;
- `convergence_smooth.csv` - сеточная сходимость на гладком периодическом тесте;
- `stability_cfl.csv` - поведение методов при разных числах CFL.
- `plots/*.svg` - готовые SVG-графики профилей и сходимости.

Тесты:

- `dam_break`: распад разрыва `h_L = 2`, `h_R = 1`;
- `strong_break`: более сильный разрыв `h_L = 10`, `h_R = 1`;
- `smooth`: гладкое периодическое начальное условие.

## Сборка

```powershell
cmake -S . -B build
cmake --build build --config Release
```

Запуск на Windows с многоконфигурационным генератором:

```powershell
.\build\Release\shallow_water_compare.exe
```

Если CMake выбрал одноконфигурационный генератор:

```powershell
.\build\shallow_water_compare.exe
```

## Графики

Код проекта на C++ не зависит от внешних библиотек и сам пишет SVG-графики в `results/plots/`.

Для построения дополнительных PNG-графиков есть необязательный скрипт:

```powershell
python scripts\plot_results.py
```

Он требует `matplotlib`. Если его нет, CSV-файлы можно открыть в Excel, LibreOffice Calc, Python, Matlab или GNU Octave.

## Идея анализа

Для защиты удобно сравнивать методы по нескольким критериям:

- численная вязкость: насколько сильно схема размазывает разрыв;
- ошибка относительно эталонного расчета на мелкой сетке;
- сохранение полной массы;
- устойчивость при разных `CFL`;
- вычислительное время.

Ожидаемый качественный вывод:

- Rusanov самый простой и устойчивый, но наиболее диффузионный;
- HLL обычно дает хороший компромисс между устойчивостью и резкостью фронтов;
- Roe лучше разрешает волны, но сложнее и чувствительнее, поэтому добавлена энтропийная поправка.
